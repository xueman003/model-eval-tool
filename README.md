# 模型结果自动评测工具

面向模型运营的本地化 Excel 评测分析与准召统计工具。

这是一个面向模型运营日常评测流程的本地网页小工具。它可以把 Excel 中的原始 JSON 评估结果自动拆成结构化字段，并根据用户选择的模型分列、正确结果列和正例定义，生成分层统计、precision、recall、F1 等准召指标。

工具采用 local-first 设计，数据只在本机处理，不依赖外部 API，适合处理包含样本 ID、模型输出、人工标签等字段的模型评测表。

## 核心卖点

- **完全本地化处理**：上传、解析、统计、导出都发生在本机 `127.0.0.1` 服务中，不会把评测数据发送到外部平台。
- **自动拆分 JSON 结果**：支持把模型输出中的 JSON 字段拆成 Excel 列，例如 `prob_1`、`prob_2`、`reason` 等。
- **支持列表字段展开**：如果 JSON 中包含列表，会自动展开为多列，例如 `items` 会拆成 `items_1`、`items_2`、`items_3`。
- **兼容失败样本**：空输出、运行失败、非法 JSON 不会中断整批处理，会保留原行并用 `_json_parse_status` 标记。
- **灵活定义正例**：支持自定义一个或多个正例取值，例如 `target_a`、`target_b`，适配多分类评测结果。
- **自动计算准召指标**：按 `0.01` 阈值粒度生成 precision（准确率）、recall（召回率）、F1，并统计 TP、FP、FN，方便快速判断模型效果。
- **支持模型分分层统计**：按 `0.01` 分数区间统计样本量、正例数、正例率和召回占比，帮助定位最佳阈值区间。
- **支持只导出拆分明细**：拆分 JSON 后可直接下载结构化明细，便于用户自行做统计分析。
- **保护长 ID 格式**：导出时会将 `ObjectID`、`对象ID` 等列设置为文本格式，避免长数字 ID 后几位被 Excel 改写。
  注意：如果源 Excel 文件中长 ID 已经被保存为数字格式，Excel 可能在源文件阶段就已经丢失精度。工具会在读取和导出阶段尽量按文本保留 ID，但无法恢复源文件中已经被 Excel 改写的数字。
- **支持多工作表 Excel**：上传后可选择要处理的 sheet，适配常见评测文件结构。
- **导出保护机制**：对超长文本做 Excel 单元格上限保护，并标记发生截断的列，避免导出失败。

## 适用场景

- 模型评测结果复盘
- 分类模型准召分析
- 模型输出字段拆解
- 多阈值效果对比
- 多类别目标样本召回分析
- 模型运营日常批量评测报表生成

## 使用流程

1. 上传评估结果 Excel。
2. 如果文件有多个工作表，选择要处理的工作表。
3. 选择包含原始 JSON 的列，点击“拆分 JSON 并刷新列”。
4. 选择最终结果列，也就是模型分列。
5. 选择正确结果列。
6. 如正确结果列的正例不是 `1`、`是`、`true`、`命中` 等常见值，在“正例取值”里填写精确文本。
7. 如果有多个正例取值，可用逗号、中文逗号或换行分隔，例如 `target_a,target_b`。
8. 如果只需要拆分后的明细，可直接点击“下载拆分明细”。
9. 如果需要准召统计，继续选择分数方向，生成统计并下载结果。

## 输出内容

下载的 Excel 包含：

- `拆分明细`：原始列 + JSON 拆出的字段 + `_json_parse_status`
- `统计概览`：有效样本数、正例数、选中的列
- `分层分布`：按 `0.01` 模型分区间统计样本数、正例数、正例率、召回占比，并按分数区间倒序展示
- `阈值准召`：按 `0.01` 阈值统计 precision、recall、F1，并按阈值倒序展示

空输出、运行失败文本、非法 JSON 不会中断处理，会在 `_json_parse_status` 中标记为 `empty` 或 `invalid_json`。

## 指标口径

工具中的“正例”由用户选择的 `正确结果列` 和填写的 `正例取值` 决定：

```text
真实正例 = 正确结果列的值命中正例取值
```

如果选择“分越高越像正例”：

```text
预测正例 = 模型分 >= 阈值
```

如果选择“分越低越像正例”：

```text
预测正例 = 模型分 <= 阈值
```

## 指标说明

- **Precision（准确率）**：模型判为正例的样本里，实际也是正例的比例。适合看模型“判出来的结果准不准”。
- **Recall（召回率）**：所有真实正例里，被模型成功召回的比例。适合看模型“有没有漏掉目标样本”。
- **F1**：precision 和 recall 的综合指标，适合在准确率和召回率都重要时做整体比较。
- **TP**：模型判为正例，实际也是正例。
- **FP**：模型判为正例，但实际不是正例。
- **FN**：模型没有判为正例，但实际是正例。

指标计算方式：

```text
precision = TP / (TP + FP)
recall    = TP / (TP + FN)
F1        = 2 * precision * recall / (precision + recall)
```

## 保护机制

- 支持选择 Excel 工作表。
- 支持多个正例取值，例如 `target_a,target_b`。
- 导出 Excel 前会自动处理超过单元格上限的超长文本。被截断的行会新增 `_excel_truncated_columns` 标记，记录哪些列发生过截断。

## 隐私说明

本工具默认只在本机运行：

```text
http://127.0.0.1:8765
```

`127.0.0.1` 表示本机地址。工具不会主动连接外部 API，也不会上传用户数据。评测文件会临时保存在本地 `uploads/` 目录，生成结果保存在本地 `outputs/` 目录。
