# 模型结果自动评测工具项目介绍

## 项目简介

模型结果自动评测工具是一款面向模型运营场景的本地化 Excel 评测分析工具。

在日常模型评测中，模型输出结果往往以 JSON 字符串的形式存储在 Excel 单元格中，人工拆字段、筛选结果、计算 precision、recall、F1 和分层统计会消耗大量重复时间。这个工具的目标，是把这套高频、重复、容易出错的分析流程沉淀成一个可复用的小工具。

工具支持本地运行，用户只需要上传 Excel，选择模型输出列、模型分列和正确结果列，即可自动完成 JSON 拆分、异常样本标记、分层统计和准召指标计算，并导出结构化 Excel 报告。

## 项目亮点

### 1. 完全本地化运行

工具运行在本机地址：

```text
http://127.0.0.1:8765
```

评测文件只在用户电脑本地处理，不依赖外部 API，不会上传到第三方服务。适合处理包含样本 ID、人工标签、模型输出等信息的评测文件。

### 2. 自动拆分 JSON 输出

工具可以把 Excel 单元格中的 JSON 字符串自动拆成独立列。

例如原始输出：

```json
{"prob_1": 0.82, "prob_2": 0.18, "reason": "示例说明"}
```

会被拆分为：

```text
prob_1
prob_2
reason
```

这可以减少人工复制、粘贴、写公式或脚本处理的工作量。

### 3. 支持列表字段展开

如果 JSON 中包含列表字段，工具会自动展开成多列。

例如：

```json
{"items": ["item_a", "item_b", "item_c"]}
```

会被拆分为：

```text
items_1
items_2
items_3
```

如果不同样本中的列表长度不同，工具会按最长列表生成列，较短列表对应的后续列留空。

### 4. 兼容异常样本

在实际评测任务中，可能会出现：

- 空输出
- 运行失败
- 非标准 JSON
- 部分字段缺失

工具不会因为单行异常中断整批处理，而是保留原始行，并使用 `_json_parse_status` 标记解析状态，便于后续排查。

### 5. 支持自定义正例

不同评测任务对“正例”的定义可能不同。工具支持用户手动选择正确结果列，并自定义一个或多个正例取值。

例如：

```text
target_a
```

或：

```text
target_a,target_b
```

这样可以适配二分类、多分类、目标类别召回等不同评测口径。

### 6. 支持细粒度阈值准召分析

工具会按照 `0.01` 的阈值粒度计算：

- precision
- recall
- F1
- TP
- FP
- FN

阈值准召结果按阈值倒序展示，例如：

```text
>= 1.00
>= 0.99
>= 0.98
...
>= 0.00
```

这有助于快速观察不同阈值下模型表现的变化，辅助选择更合适的模型分阈值。

### 7. 支持模型分分层统计

工具会按照 `0.01` 分数区间生成分层分布，例如：

```text
0.99-1.00
0.98-0.99
0.97-0.98
...
0.00-0.01
```

每个区间会统计：

- 样本数
- 正例数
- 正例率
- 召回占比

这可以帮助分析模型分在不同区间的样本分布和效果表现。

### 8. 支持只导出拆分明细

如果用户只需要把 JSON 输出拆成结构化列，也可以在拆分后直接下载明细文件，不必配置模型分列、正确结果列和正例取值。

这适合用户希望先拿到清洗后的结构化数据，再自行做统计分析的场景。

### 9. 保护长 ID 格式

导出 Excel 时，工具会将 `ObjectID`、`对象ID` 等列设置为文本格式，避免长数字 ID 被 Excel 自动转成数字格式后丢失精度。

如果源 Excel 文件中长 ID 已经被保存为数字格式，Excel 可能在源文件阶段就已经丢失精度。工具会在读取和导出阶段尽量按文本保留 ID，但无法恢复源文件中已经被 Excel 改写的数字。

例如：

```text
7651543994911984905
```

会尽量以文本形式保留，避免后几位被改写。

### 10. 支持多工作表 Excel

如果上传的 Excel 中包含多个 sheet，工具会提供工作表选择能力，用户可以选择要处理的具体 sheet。

### 11. 支持导出结构化报告

处理完成后，工具会导出一个新的 Excel 文件，包含：

- `拆分明细`
- `统计概览`
- `分层分布`
- `阈值准召`

结果既可以直接用于复盘，也可以继续作为后续分析材料。

## 使用方法

### 1. 下载项目

从 GitHub 下载项目 ZIP 并解压到本地。

### 2. 启动工具

根据系统选择对应启动文件：

```text
macOS：双击 start.command
Windows：双击 start.bat
Linux：运行 start.sh
```

首次启动时，工具会自动检查并安装依赖。

启动成功后，会自动打开浏览器。如果没有自动打开，可以手动访问：

```text
http://127.0.0.1:8765
```

### 3. 上传并分析 Excel

操作流程：

1. 上传评测结果 Excel。
2. 如果文件有多个工作表，选择要处理的 sheet。
3. 选择包含模型 JSON 输出的列。
4. 点击“拆分 JSON 并刷新列”。
5. 如果只需要拆分后的数据，直接点击“下载拆分明细”。
6. 如果需要准召统计，继续选择模型分列。
7. 选择正确结果列。
8. 填写正例取值。
9. 选择分数方向。
10. 点击“生成统计与 Excel”。
11. 查看页面统计或下载结果文件。

## 指标说明

### Precision

模型判为正例的样本中，实际也是正例的比例。

```text
precision = TP / (TP + FP)
```

适合观察模型“判出来的结果准不准”。

### Recall

所有真实正例中，被模型成功召回的比例。

```text
recall = TP / (TP + FN)
```

适合观察模型“有没有漏掉目标样本”。

### F1

precision 和 recall 的综合指标。

```text
F1 = 2 * precision * recall / (precision + recall)
```

适合在准确率和召回率都重要时做整体比较。

## 项目价值

这个项目的价值不在于复杂技术堆叠，而在于把模型运营中高频、重复、容易出错的评测分析流程产品化。

它将原本需要人工拆字段、写公式、筛选数据、计算指标的流程，整合为一个可视化、本地化、可复用的小工具，降低了模型评测分析的操作门槛，也提升了日常复盘效率。
